# Guidelines

This file provides guidance when working with code in this repository.

## Project Overview

This is a **playground** for testing new features, experimental purposes, and Minimum Viable Product. This repository typically does not require unit testing or complex design; it focuses on rapid implementation.

Each subdirectory is distinguished by its purpose, and the naming follows the principles of clarity and distinctiveness. Use README.md in the directory to describe the purpose of each directory in one or two sentences, but environment setup should be documented in detail, including environment setup, dependencies, commands, and special considerations.
